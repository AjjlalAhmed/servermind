Features

Everything you'd SSH in for — just ask.

Plain-English control over your whole stack, with a live dashboard and guardrails that keep it safe.

Chat your infrastructure

“Why is nginx down?” · “Restart api-prod” · “How much disk is left?” It picks the right tools, runs them, and explains what it finds.

Live dashboard

CPU, memory, disk, uptime, PM2 processes and service health — at a glance, with sparklines and a status pulse, auto-refreshing.

Safe by design

A strict read-only command whitelist, no raw shell, and a server-enforced “arm” switch required before any change touches your box.

Locked down

How it works

From zero to chatting in three steps.

01

Install

Run one command on your VPS. It sets up Bun, PM2, and ServerMind automatically.

02

Set up

The wizard walks you through a password, a TOTP QR scan, and picking your AI backend.

03

Chat

Open your dashboard and ask your server anything. Arm mutations when you want to make changes.

Security

Your server. Your keys.
Your data.

ServerMind runs on your own machine. Your AI keys stay in your .env, your logs never leave the box, and every command the AI can run is a vetted, read-only-by-default tool — there's no raw shell to escape into.

  • Argv-only execution — command injection is structurally impossible
  • Server-enforced “arm” gate before any restart or change
  • Password + TOTP 2FA, HttpOnly sessions, brute-force lockout
  • Strict Content-Security-Policy, fully self-hosted assets
shell allowlist
# read-only by default — no rm, no writes, no network
$ df -h            ✓ allowed
$ systemctl status ✓ allowed
$ rm -rf /          ✗ REJECTED
$ cat /etc/shadow   ✗ REJECTED

# mutations need you to arm them first
$ pm2 restart api   ⚠ DISARMED — flip the switch
